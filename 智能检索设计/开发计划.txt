# 智能检索系统 - 开发计划

> 创建日期：2026-03-21
> 最后更新：2026-03-21（W02 完成）

---

## 一、项目概述

基于 Agent + 结构化搜索 + 知识检索 + 实时查询的智能检索系统。
用 Agent 把已有业务能力编排起来，形成自然语言交互入口。

---

## 二、里程碑规划

### M1：搜索 MVP（第 1-5 周）
- 目标：跑通"用户提问 → 意图识别 → 结构化搜索 → 生成回答"的核心链路
- 包含：W00 ~ W08
- 交付物：可演示的搜索推荐 Agent

### M2：知识 + 实时能力接入（第 3-6 周，与 M1 部分并行）
- 目标：Agent 具备 FAQ 问答和实时状态确认能力
- 包含：W09、W10
- 交付物：FAQ 可回答、实时库存/营业状态可确认

### M3：多轮会话与混合链路（第 6-7 周）
- 目标：支持多轮对话、条件追加、意图切换、混合链路
- 包含：W11
- 交付物：多轮会话可用，6 条典型链路跑通

### M4：生产就绪（第 7-10 周）
- 目标：同步补偿、监控评估、联调上线
- 包含：W12 ~ W14
- 交付物：可灰度上线，可观测，可回滚

## 四、关键依赖与风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|---------|
| ES 集群申请周期长 | W03 阻塞 | 第 1 周提前申请，本地先用 Docker ES 开发 |
| LLM API 配额不足 | 全链路受限 | 提前评估调用量，申请配额 |
| 知识库内容不全 | W09 效果差 | 先用现有 FAQ 数据跑通链路，内容逐步补充 |
| 单人开发时主线支线无法并行 | 日历时间拉长 | 优先主线 W00-W08，支线延后 |
